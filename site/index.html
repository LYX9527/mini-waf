核心功能

专为小团队和个人开发者设计，无需运维专家即可部署企业级 WAF。

域名路由

支持按域名 + 路径前缀双维度精准路由。同一端口同时保护多个域名的不同后端服务，无需额外 Nginx 层。

攻击拦截

内置 SQL 注入、XSS、目录穿越等规则，支持自定义关键字和正则，热重载无需重启。

IP 黑白名单

精细化 IP 准入控制，支持手动添加封禁/放行，配合 GeoIP 数据库实现国家级流量管控。

Nginx 可视化管理

通过控制台直接编写和下发 Nginx 配置，保存后自动 reload，无需 SSH 登录服务器。

实时访问日志

完整记录每一次请求的 IP、路径、状态码、地理位置，攻击拦截日志单独归档，支持快速溯源。

Docker 一键部署

完整 docker-compose 编排，包含 WAF、Nginx、MySQL，一条命令启动全部服务，支持多架构镜像。

一键安装

在任意 Linux 服务器上运行以下命令，30 秒内完成部署。

一键安装命令
bash <(curl -sSL https://wf.yltf.org)
1

系统要求

已安装 Docker 和 Docker Compose，任意 Linux 发行版均可，推荐 Ubuntu 22.04 / Debian 12。

2

运行安装脚本

脚本会自动下载配置文件，生成随机安全密钥，并启动全部容器（WAF + Nginx + MySQL）。

3

访问控制台

安装完成后访问 http://<服务器IP>:49777，完成首次初始化设置管理员账号。

4

配置路由规则

在「站点管理」中添加域名和路径路由，将流量代理到你的内部服务。

端口说明

安装后默认暴露以下端口，可在安装时自定义。

端口 用途 说明
49888 WAF 代理入口 所有需要保护的流量都应从此端口进入，映射到 Nginx 经由 WAF 过滤
49777 管理控制台 WAF 管理后台，建议通过防火墙限制只允许可信 IP 访问

使用说明

了解如何通过 Mini-WAF 保护你的内部服务。

# 典型部署架构 外部用户 --> 服务器公网 IP:49888 --> WAF (48080) | +--- 路由匹配 (Host + 路径) ---+ | | api.example.com / web.example.com / | | 后端服务 :3000 前端服务 :8080 # 如何让域名流量进入 WAF # 方式一: 域名 DNS 解析到服务器 IP，Nginx 监听 80/443 # Nginx 将请求代理到 WAF 的 48080 端口 # 方式二: 直接将 49888 端口开放给公网，通过域名访问
A

添加站点路由

在「站点管理」中新增路由条目：
选填「域名限制」（如 api.example.com），填写「路径前缀」（如 /），填写上游地址（如 127.0.0.1:3000）。

B

配置 Nginx

在「Nginx 管理」中为你的域名添加 server 块，将 80/443 端口的请求 proxy_pass 到 http://mini-waf:48080，保存后自动生效。

C

设置防护规则

在「安全规则」中启用内置规则或自定义规则，在「IP 管理」中添加黑名单，支持热重载无需重启。

D

监控日志

在「攻击日志」和「访问日志」中实时查看流量情况，可按 IP、时间、规则等筛选定位异常请求。

常见问题

如何区分不同域名指向不同的后端服务？
在「站点管理」添加路由时，填写「域名限制」字段（如 api.example.com 或 *.example.com）。WAF 会读取请求的 Host 头进行匹配，多条路由可以共用相同路径前缀但绑定不同域名。
如何将已有域名的流量导入 WAF？
推荐方式：在「Nginx 管理」中为域名创建配置，监听 80/443 端口，将请求 proxy_pass 到 http://mini-waf:48080，并设置 proxy_set_header Host $host 以传递原始域名。
管理控制台 49777 端口需要对外开放吗？
不建议对公网开放。推荐通过服务器防火墙（如 ufw、iptables）限制 49777 端口只允许你的管理 IP 访问，或通过 SSH 隧道 (ssh -L 49777:127.0.0.1:49777 user@server) 进行本地转发。
如何更新 Mini-WAF 到最新版本？
进入安装目录后执行：docker compose pull && docker compose up -d，数据和配置会自动保留。
Nginx 管理报错「无法连接 Docker Socket」怎么办？
确认 docker-compose.yml 中 mini-waf 服务挂载了 Docker Socket：- /var/run/docker.sock:/var/run/docker.sock，并确保 Docker 守护进程的 socket 文件存在。
